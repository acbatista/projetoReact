Ciclo de Vida

Quatro fases de um componente React

  1. Inicialização
  2. Montagem
  3. Atualização
  4. Desmontagem

Ciclo de Vida

1 - Inicialização

Nesta fase, o componente React se prepara para sua inicialização, configurando os estados iniciais e props padrões se houverem.

  1. constructor
  2. instance variables

Ciclo de Vida

2 - Montagem

Depois de preparar com todas as necessidades básicas, estado e props, o nosso Componente React está pronto para ser montado no DOM.

  1. componentWillMount - É executado quando o componente estiver prestes a ser montado.
    Todas as coisas que você deseja fazer antes do componente ser montado, devem ser definidas aqui. Este método é executado uma vez em um ciclo de vida de um componente e antes da primeira renderização.
  2. render
  3. componentDidMount - É executado depois que o componente foi montado.
    Este método é executado uma vez em um ciclo de vida de um componente e será após a primeira renderização. As chamadas de API (request etc) devem ser feitas sempre no método componentDidMount.

Ciclo de Vida

3 - Atualização

Esta fase começa quando o componente React já foi montado e recebe novas atualizações (Externas ou internas).

O componente pode ser atualizado de duas maneiras, através de uma "pertubação externa" (do envio de novas props) ou de uma atualização interna, a atualização do seu próprio estado.

  1. componentWillReceiveProps - é executado quando as props mudaram.
    Às vezes, o estado depende das props, portanto, sempre que as props mudam o estado também deve ser sincronizado. Este é o método onde deve ser feito isso.
  2. shouldComponentUpdate - Diz ao React que, quando o componente recebe novas props ou estado, o React re-renderiza ou pode ignorar a renderização do componente.
    O Componente foi atualizado?” Portanto, esse método deve retornar verdadeiro ou falso, e então o componente seria re-renderizado ou ignorado. Por padrão, esse método retorna verdadeiro.
  3. componentWillUpdate - É executado somente quando shouldComponentUpdatedevolver true.
    Este método é usado apenas para preparar a próxima renderização, semelhante ao componentWillMount ou ao construtor.
  4. render
  5. componentDidUpdate - É executado quando o componente foi atualizado no DOM.

Ciclo de Vida

4 - Desmontagem

Nesta fase, o componente não é mais necessário e será desmontado do DOM.

  1. componentWillUnmount - Este método é o último método no ciclo de vida. É executado imediatamente antes de o componente ser removido do DOM.

Context (Legado React < 16.3)

Permite passar dados pela árvore de componentes sem precisar informar as Props para cada nível.

Em um aplicativo React normal, os dados são passados de cima para baixo (top-down, pai para filho) via props, mas isso pode ser complicado para certos tipos de props (por exemplo, preferência de localidade, internacionalização, tema de UI) que são exigidos por muitos componentes. Contexto fornece uma maneira de compartilhar valores como esses entre componentes, sem ter que passar explicitamente uma Props em todos os níveis da árvore.

Context É UMA VARIÁVEL GLOBAL (quasi-global)!!

Refs (Legado React < 16.3)

Permite acessar o DOM ou o React Element que foi criado no método render

No fluxo de vida normal de um componente React, as Props são a única maneira de os componentes pai interagirem com seus filhos. Para modificar um filho, você o re-renderiza com novos Props. No entanto, existem alguns casos em que você precisa modificar obrigatoriamente um filho de forma mais direta (o proprio elemnto). O filho a ser modificado pode ser uma instância de um componente React ou pode ser um elemento DOM. Para ambos os casos, o React fornece esse acesso.

  1. Gerenciar FOCUS em inputs
  2. Forçar seleção de texto
  3. Controlar um componente de mídia (play, pause, volume, etc)
  4. Executar animações imperativas
  5. Integrar com bibliotecas de terceiros que dependem de um elemento HTML (DOM)

Evite usar o Refs, use APENAS QUANDO NÃO TIVER OUTRA OPÇÃO!
