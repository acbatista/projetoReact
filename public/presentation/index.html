ReactJS

Biblioteca JavaScript para construir interfaces de usuário.

Criada pela equipe do Instagram (antes do Facebook comprar) entrou para a lista de projetos open-source do Facebook, impulsionando mais ainda sua adoção. O grande “boom” do React é o Virtual-DOM.

O React foi pensado em como podemos componentizar nossos elementos, de forma que eles sejam independentes, mas reutilizáveis.

Diferentemente do Backbone.JS, AngularJS, Ember e outros que fornecem todos os componentes necessários para uma aplicação front-end, o React se preocupa apenas com a parte de view.

Virtual DOM

O Virtual DOM é um árvore virtual (JavaScript puro) que representa o estado de cada componente na tela.

Quando algum componente precisa ser atualizado, o React não renderiza todo o DOM inteiro novamente. Ao invés disso ele descobre exatamente quais elementos precisam ser atualizados e faz a menor quantidade possível de mudanças no DOM.

No React Native, o Virtual DOM compara o novo componente com o estado antigo do DOM Virtual e envia para Bridge (barramento utilizado para comunicação entre o motor JavaScriptCore e o Renderizador) somente as informações que foram modificadas.

JSX

A parte MAIS esquisita no primeiro contato com a biblioteca React.

O JSX é uma especificação de sintaxe para escrever JavaScript como se estivéssemos escrevendo XML.

JSX pode ser utilizado com qualquer código JavaScript, não só React. 

class Site {};

var site = <Site url="http://www.google.com" nome="Google" />;
var site = Site({url:"http://www.google.com", nome:"Google"});

O XML é convertido para a chamada da classe. Os atributos da tag viram propriedades do objeto.

TypeScript

É um superset de JavaScript que compila para JavaScript simples.

TypeScript fornece classes, interfaces e tipagem estática.

Uma das grandes vantagens do TypeScript é permitir que os IDEs proporcionem um ambiente mais rico para detectar erros comuns enquanto você digita o código.

TypeScript - Por quê?

Para ter JavaScript fortemente tipado;

Você aproveita o seu conhecimento da POO para escrever aplicativos JavaScript melhores;

Equipes se beneficiarão do uso do TypeScript, visto que os erros podem ser detectados enquanto o código é digitado.

TypeScript faz a verificação de erros durante a compilação e permite gerar Javascript para diferentes versões do EcmaScript sem que você precise necessariamente se preocupar com isso.

Assim, em um projeto que você vai usar muito JavaScript, adotar o TypeScript pode trazer um ganho de produtividade e resultar em um produto mais robusto.
